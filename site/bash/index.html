A shell é um intérprete de comandos que fornece uma linha de comando(prompt) que faz de interface interativa e não interativa entre o utilizador e o sistema operativo.

Nessa linha de comando é onde inserimos os comandos. Eles são interpretados pela shell e então enviados como instruções para o sistema operativo (existe recurso grafico a interface da linha de comando via GNOME ou KDE).

É possivel colocar variados comandos criando um ficheiro shell script para ser executado consecutivamente, como um programa.

A capacidade interpretativa do shell oferece muitos recursos técnicos: Por exemplo, o shell possui um conjunto de metacaracteres de correspondência para ficheiros que podem gerar nomes de ficheiros.

A shell consegue redirecionar informações de entrada e saída, executar operações em segundo plano, libertanto assim a linha de comando para executar outras tarefas.

varios tipos de shells foram desenvolvidos para o sistema linux. mas apenas é necessario selecionar um tipo shell para realizar suas operacoes básicas de trabalho.

  • bash, Bourne Again
  • ksh korn shell
  • tcsh, versao melhorada do shell C utilizado muito no sistema BSD. csh é um link para tcsh
  • zsh z shell

A interface da linha de comando do Linux consiste em uma única linha na qual o utilizador insere os comandos com algumas opções e argumentos.

Noções Básicas sobre a linha de comando - prompt # $

A maneira tradicional de operar com um computador Linux – especialmente em ambientes servidor – é na linha de comando. A linha de comando apresenta o prompt do shell indicando que o sistema esta pronto para receber instrucoes. Normalmente, o prompt termina com o caractere $ indica que o utilizador comum esta activo no sistema. Enquanto termina com o caractere #, indica ser o super utilizador, o root. o formato comum da prompt informa o nome do utilizador seguido do hostname: [user@hostname].

Root - O Super Utilizador

É o unico utilizador capaz de realizar todo tipo de operacoes no sistema. Por questoes de seguranca e privacidade, todas as actividades e ficheiros têm um utilizador responsavel representando por um “dono”. Cada "dono" pode interferir apenas nas tarefas e ficheiros que lhe pertencem. O utilizador root têm a caracteristica de interferir em qualquer tarefa ou ficheiro sem ser dono do mesmo.

Shell Bash

Uma interpretacao simples do funcionamento da shell: comando -> shell -> kernel -> hardware

No ambiente de linha de comando, o shell vai ser o programa que faz a intermediaçao entre o utilizador e os recursos do computador, oferece um ambiente de programaçao em tempo real para executar tarefas.

O shell padrao usado na maioria das distribuiçoes Linux e o BASH (Bourne Again Shell). Este shell oferece diversos comandos, internos e externos mas só os internos são embutidos na shell.

Um dos comandos embutidos mais simples e o echo, que simplesmente "ecoa" um conteudo no ecra:

[user@hostname] echo Linux Essentials
Linux Essentials
[user@hostname]

Apos escrever o comando echo Linux Essentials, o comando deve ser executado pressionando a tecla Enter.

A resposta do echo sera apresentada logo abaixo do comando. Para evitar que o comando echo inicie espaçamento de nova linha no final da saıda, utiliza-se a opcao -n.

[user@hostname] echo -n Linux Essentials
Linux Essentials[user@hostname]

Para dar uso a caracteres especiais, devemos colocar a opcao -e. Por exemplo, para criar um novo espaçamento de linha entre as duas palavras:

[user@hostname] echo -e "Linux\nEssentials"
Linux
Essentials

se necessario para vizualisar sem caracteres especiais colocamos -E

[user@hostname] echo -e "Linux\nEssentials"

Definição dos Comandos Internos e Externos

  • Comandos Internos São comandos que estão localizados dentro do interpretador de comandos (normalmente uma shell) e não no disco. Eles são carregados na memória RAM do computador junto com o interpretador de comandos.

cd, exit, echo, bg, fg, source, help

Quando executa um comando, o interpretador de comandos verifica primeiro se ele é um Comando Interno caso não seja é verificado se é um Comando Externo.

  • Comandos Externos São comandos que estão localizados no disco. Os comandos são localizados no disco usando a ordem encontrada na variavel $PATH e executados assim que encontrados.

Pathname é o caminho usado para localizar ficheiros/comandos executáveis.

Definir Variavel

As variaveis usadas na shell sao parecidas às de outras linguagens de programacao. Uma variavel e um nome que guarda um valor, que pode ser letras ou numeros. Nomes de variaveis sao limitados a caracteres alfanumericos, ou seja, pode conter letras e numeros, mas deve sempre comecar com uma letra.

Variaveis podem ser criadas por utilizadores comuns ou pre-definidas pelo sistema.

Para criar ou modificar uma variavel, nao devem ser usados espacos:

lpi="Linux Professional Institute"

acabamos de definir que a variavel $lpi tem a seguinte frase, Linux Professional Institute.

Se existir espaços em branco no conteudo da variavel, é importante usar as entreaspas duplas ou simples para nao confundir a interpretacao da shell. O valor de uma variavel pode ser apresentado colocando o sinal $ a frente do nome da variavel:

$ echo $lpi
Linux Professional Institute

Variaveis Globais - env

Todas as variaveis globais sao escritas em letras maiusculas. Em ambientes Unix existe a regra do case-senstive, diferença entre letras maiúsculas e minúsculas.

Portanto, escrever PATH é diferente de path.

As variaveis pre-definidas – tambem conhecidas de globais ou variaveis de ambiente – sao usadas por programas para obter configuraçoes importantes do sistema. O proprio shell utiliza as variaveis globais para definir diversas configuraçoes do sistema.

O shell interpreta a primeira palavra fornecida como um comando. A localizaçao para o comando precisa ser fornecida, a menos que este esteja localizado em um das pastas indentificadas na variavel global PATH.

Se o programa estiver na pasta atual e fora das pastas indentificadas em PATH, seu nome deve ser precedido por ./, por exemplo ./script.sh.

O conteudo da variavel PATH pode ser exibido com o comando echo:

    echo $PATH
    /usr/local/sbin:/usr/local/bin:/usr/bin

A localizacao especıfica de um comando pode ser determinado com outro comando, which

$ which man
/usr/bin/man

A partir da saıda do comando which man é possıvel determinar que o comando man está localizado no /usr/bin/man.

Historico de comandos na shell

Shell BASH mantem uma lista dos comandos, chamada de lista do historico dos comandos previamente inseridos. para exibir cada comando inserido, por ordem decrescente pressionando a tecla seta para cima. ja a tecla seta para baixo move pela ordem inicial. Pode recuperar qualquer comando previamente digitado movendo as setas. A capacidade de exibir novamente um comando anterir é pratico desta forma nao necessita de escrever o comando inteiro novamente.

Eventos no historico

o utilitario do historico mantem um registro dos comandos mais recentes que foram inseridos. os comandos sao numerados começando em 1, existe um limite para o numero de comandos lembrados - o padrao é 500. todo historico de comandos e exibido com o comando history.

cada comando fica enumerado na lista. trata como um evento que descreve uma accao tomada. um comando foi executado. o evento mais recente tem o numero mais alto.

atalhos descricao
CTRL+p move para o anterior evento na lista de historico
CTRL+n move para o proximo evento na lista de historico

Configurar Historico: HISTFILE e HISTSIZE

O numero total de eventos guardados pelo sistema é mantido numa variavel especial chamada HISTSIZE, o valor padrao é 500 eventos. podemos alterar o valor alterado o numero da variavel.

echo $HISTSIZE # mostra o valor 500
export HISTSIZE=10

Os eventos do historico sao armezanos num ficheiro na pasta inicial do utilizador cujo o nome e mantido numa variavel de sistema chamada HISTFILE. por padrao é um ficheiro oculto chamado .bash_history

echo $HISTFILE # localiza o ficheiro .bash_history
/home/utilizador/.bash_history

para limpar o registo do historico é usada a opcao -c: history -c

variaveis globais sao criadas no carregamento do sistema e exportadas para que esteja disponiveis em todas as sessoes futuras da shell.

Sequencias de comandos

A maioria das tarefas dependem da execucao de varios comandos. Para executar tres comandos em sequencia. indenpente do resultado de cada um:

comando1 ; comando2 ; comando3

executar o comando seguinte se o anterior foi bem sucedido (retornou 0)

apt-get update && apt-get upgrade

executar o comando seguinte apenas se o anterior nao for bem sucedido (retornar diferente de 0)

comando1 || comando2 || comando3

todos os comandos obedecem a estas sintaxes. no entanto cada comando possui uma finalidade e opcoes diferentes. É necessario consultar a sua documentacao.

globbing/englobamento

É uma operação que combina metacaracteres como * e ? com partes de nomes de ficheiros para a formação de todas as combinações possíveis desses nomes de ficheiros.

Esses nomes expandidos em combinações possíveis são passados como parâmetros para comandos do Shell que não poderiam lidar com expressões regulares em seu modo nativo.

Fazer globbing é muito parecido com expressões regulares, mas NÃO é expressões regulares.

ls -l [ab]*
ls -l [a-c]*
ls -l [^ab]*
ls -l {b*,c*,*est*}

combinar varios caracteres
o asterico (*) faz referencia a nenhum ou a mais caracteres


combinar exactamente um caracter
?


alcance de 1 a 10 por exemplo
[]


combinar simbolos shell
raros casos o nome do ficheiro tem um metacaracter. para transcrever usa o \ ou aspas duplas
ls perguntas\?

ls "perguntas?"

ficheiros ou pastas que possuem nomes com caracteres com espaço em branco. Devem ser escritos com \ ou entre aspas

ls Meus\ Documentos
ls "Meus Documentos"



gerar padroes
embora nao seja uma operacao de expansao de ficheiros o {} muitas vezes e util para gerar nomes de ficheiros ou pastas. Nao existe verificacao de correspondecia dos nomes ja criados. cada palavra e separada por virgula SEM espaços em branco

echo se{rafim,simbra,ca}
