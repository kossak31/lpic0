FHS do Linux

  • boot : ficheiros necessários para inicialização do kernel do Linux
  • bin : comandos gerais para utilizadores
  • dev : ficheiros de dispositivos
  • etc : ficheiros de configuração do sistema
  • home : pasta inicial para ficheiros pessoais de utilizadores comuns
  • lib : ficheiros de bibliotecas
  • media : Ponto de montagem para dispositivo externo
  • proc : informações do sistema, processos
  • root : pasta inicial para root
  • sbin : comandos para administradores
  • tmp : ficheiros temporarios
  • usr : programas, codigo fonte, bibliotecas, documentos, etc.
  • var : ficheiros alterados com frequência, ficheiros de log

Find


# definir nivel de profundidade na pesquisa
$ find . -name "README.md" -maxdepth 1



# Pesquisa basica
# find [localização] [opcao]

# Por nome
$ find . -name "README.md"

# Por tamanho
$ find . -size 1k       # = 1k byte
$ find . -size +100c    # > 100 byte
$ find . -size +1k      # > 1k byte
$ find . -size -100c    # < 100 byte
$ find . -size -1k      # < 1k byte

# Por tipo
$ find . -type f # ficheiros
$ find . -type d # pastas
$ find . -type l # links simbolicos

# Por data modificacao
$ find . -mtime -1 # ficheiros alterados no espaço de 1 dia
$ find . -mmin -60 # ficheiros alterados no espaço de 60min
$ find . -mtime 7  # ficheiros que nao foram alterados num espaço de 1 semana
$ find . -mmin 60  # ficheiro que nao foram alterados num espaço de 60min

# Por Tempo acessido
$ find . -atime -1 # files accessed in a day
$ find . -amin -60 # files accessed in last 60 minutes
$ find . -atime 7  # files which have not accessed over a week
$ find . -amin 60  # files which have not accessed over an hour

# Por permissao
$ find . -perm a+r  # É permitida permissão de leitura
$ find . -perm 644  # codigo permissao de 644
$ find . -perm -444 # Permissão de leitura é permitida para todos os utilizadores
$ find . -perm +444 # Permissão de leitura é permitida para proprietário, grupos de proprietários e outros

# Por utilizador
$ find . -user david # ficheiros que pertencem ao david
$ find . -uid XXXXXX # ficheiro que pertencem por uid

# Regular Expressions
$ find . regex ".*git*"

Example


# Encontra ficheiros alterados no espaço de um dia
$ find . -type f -mtime -1

# Encontre ficheiros alterados nos últimos 30 minutos
$ find . -type f -mmin -60

# Localizar e excluir ficheiros que não foram acessados em 200 dias
$ find . -atime +200 -exec rm {} ¥;

Compressao Zip


#############
# gzip
#############
# ficheiro szip
$ cal > cal.txt
$ gzip cal.txt  # cria o ficheiro cal.txt.gz mas o ficheiro cal.txt é eleminado
$ gzip -c cal.txt > cal.txt.gz  # Não elemina os ficheiros originais

# comprimir uma pasta
$ mkdir tmp
$ cp cal.txt tmp
$ gzip -r tmp           # compactar pasta

# unzip
$ gzip -d cal.txt.gp # unzip
$ gzip -dr tmp           # unzip na pasta tmp
$ gunzip cal.txt.gp  # o mesmo que `gzip -d`

#############
# bzip2
#############
$ bzip2 sample.dat          # cria sample.dat.bz2
$ bzip2 -d sample.dat.bz2 # unzip


#############
# tar (com gzip)
#############
# tar [options] arquivo.tar [nomeFicheiro]
#
# options
# -c : cria
# -x : unzip
# -t : ver ficheiros de um arquivo
# -f : selecionar arquivo por nome
# -r : adicionar ficheiros ao arquivo
# -v : ver detalhes em modo verbose
# -z : gzip
# -j : bzip2

# exemplos
$ mkdir tmp
$ ls -l > tmp/ls.txt
$ cal > tmp/cal.txt
# criar arquivos tar e comprimir
$ tar cvf tmp.tar tmp           # apenas arquivar
$ tar cvfz tmp.tar.gz tmp   # arquivado e comprimido com gzip
$ tar cvfj tmp.tar.bz2 tmp  # arquivado e comprimido com bzip2
# ver detalhes nos ficheiros
$ ls -l

gzip vs bzip2 vs tar?

gzip/bzip2 vs tar?

Tar não é um formato de compressão - é uma maneira de combinar vários ficheiros em um único arquivo.

Bz2 e gz são formatos de compressão. as taxas de compressão são semelhantes

gzip vs bzip2?

Bz2 geralmente comprime melhor que o gz, mas é mais lento. Gz é o padrão mais comum porque é mais antigo e normalmente o bz2 não está disponível.

Hard link / Symbolic link


# ver hardlinks
$ ls -i # `ls` mostra o index de node(inode)

# criar hard links
$ touch test.txt                # criar ficheiro test.txt
$ ln test.txt test.hard # criar ligacao hard link
$ ls -i             # verificar numeros de inode

# criar symbol links
$ touch symbol.txt
$ ln -s symbol.txt symbol.sym
$ ls -l

# ver as diferenças de uma ligação hard e symbol
$ touch {hard,symbol}.txt           # cria hard.txt e symbol.txt
$ echo "ligacao hard" > hard.txt && echo "ligacao symbol" symbol.txt
$ ln hard.txt hard.hard                 # cria hard link
$ ln -s symbol.txt symbol.sym   # cria symbol link
$ rm {hard,symbol}.txt                  # eleminar ficheiros .txt
$ cat hard.hard                                 # existe?
$ cat symbol.sym                                # existe?

Hardlink vs Symlink?

Um Hardlink não é um atalho para um ficheiro, é uma entrada de ficheiro direcionado para o mesmo inode. Pode alterar o nome do outro ficheiro, Hardlink continua apontar para o mesmo inode ...

Por outro lado, um Symlink está realmente apontando para outro caminho (um nome de arquivo); Ele resolve o nome do arquivo toda vez que você acessá-lo através do link simbólico.

Redirect

# resultado redirecionado
$ ls > redirect.txt     # the result of `ls` command would be written to redirect2.txt
$ cat redirect.txt
$ ls -i1 >> redirect.txt    # adds the result to the existing file

# redirect error
$ ls -222 2> error.txt  # 2> writes error log
$ ls -333 2>> error.txt # 2>> adds error log to the existing file

# redirect both result and error
# .   = current file
# aaa = not-existring file
$ ls . aaa > ls.log 2>&1 # wrtites both the result and error log

# tips
# by combining `cat` and redirect, you can join the result of
# multiple files contents
$ cat apple.txt banana.txt > banapple.txt

# input redirect
# `tr a-z A-Z` replaces all small caps to Capital caps, but
# this command cannot have file as an argument.
# By usring input `<`, a content of file can be sent to commands.
$ tr a-z A-Z  < error.txt

Pipe


$ ls -l > ls.txt
$ tr a-z A-Z < ls.txt

# By using pipe, above process can be written in one line
$ ls -l | tr a-z A-Z

# If you want to send the result both to a file and other command
# then use `tee`
$ ls -l | tee result.log | less

Permission

Owner (user/group)


# changes owner
$ chown alice sample
$ chown -R alice sample # changes all files in the directory

# changes group
$ chgrp develp tmp
$ chgrop -R develop tmp

# changes owner and group at the same time
$ chown alice:develop tmp

Accessible Types

  • Permission
    • r: read(4)
    • w: write(2)
    • x: execute(1)
  • Target Segment
    • owner
    • group to which the owner belongs
    • other users
  • Permission Counters
    • 7 : 4 + 2 + 1 = r + w + x
    • 5 : 4 + 0 + 1 = r + x
    • 4 : 4 + 0 + 0 + r

$ touch sample.txt
$ ls -l sample.txt
-rw-r--r--  1 username  group  0  1 10 10:11 sample.txt
# user   = can read and write
# group  = can read only
# others = can read only

# changes permission of sample.txt
$ chmod 755 sample.txt
$ ls -l sample.txt
-rwxr-xr-x  1 username  group  0  1 10 10:11 sample.txt

# other options
# - target  : u, g, o
# - control : +, -, =
# - type        : r, w, x
#
$ chmod ug+rx sample.txt
$ ls -l sample.txt
-rwxr--r--  1 username  group  0  1 10 10:11 sample.txt

# shows executed commands history
$ history
$ history -c            # deletes history
$ history 5             # shows the 5 latest history
$ echo $HISTSIZE    # shows the size of saved history
$ !cat                      # executes the latest command begin with `cat`
$ !732                      # executes the history with the number
$ !!                            # executes the last command in history
$ !-3                       # executes the 3 before command in history
$ ^cat^ls         # executes the latest command, after replacing `ls` command with `cat`

Basic Syntax

Parameters

  • $# : the number of parameters
  • $@ : todos os parametros
  • $1 : primeiro parametro
  • $2 : segundo parametro
  • $3 : terceiro parametro...

Condition Expression

  • Tipo de ficheiro ( retorna true quando... )
    • -e : ficheiro existe
    • -f : tipo file
    • -d : pasta
    • -r : ficheiro existe e permite ler
    • -w : ficheiro existe e permite escrever
    • -x : ficheiro existe e permite executar
    • -L : é link simbolico
  • Numbero ( retorna true quando... )
    • A -eq B : A é igual ao B
    • A -ge B : A é igual/maior que B
    • A -gt B : A é maior que B
    • A -le B : A é igual ou menor que que B
    • A -lt B : A é menor que B
    • A -ne B : A não é igual B
  • String ( returna true quando... )
    • -z A : o tamanho de A é 0
    • A = B : A igual a B
    • A != B : A não é igual a B
  • Boleano (retorna true quando... )
    • ! A : A fica false
    • A -a B : A e B são true
    • A -o B : A ou B é true
