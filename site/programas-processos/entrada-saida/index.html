Entrada e saıda de comandos/Input Output Redirection

Processos Unix geralmente abrem tres canais de comunicaçao, que os permitem receber e emitir dados. Esses canais podem ser redirecionados de um para outros fichieros ou processos

Por padrao, o canal de entrada (chamado standard input ou stdin) teclado e os canais de saida padrao(standard output ou stdout ) e saida de erro(standard error ou stderr) sao o ecra do monitor.

valores canais
0 /dev/stdin
1 /dev/stdout
2 /dev/stderr

Redirecionamento

O fluxo de dados para redirecionar e canalizar na linha de comando acontece da esquerda para a direita.

ls -l > listagem.txt

Para redirecionar a saida padrao de um comando para um ficheiro, utiliza-se o caractere >. Caso o ficheiro existir previamente, sera sobrescrito com novos dados. Para limpar conteudo de ficheiros podemos simplesmente usar este comando: $ > ficheiro

Para adicionar os valores sem apagar o conteudo existente, usa-se >>.

redirecionar o conteudo de um arquivo para a entrada padr ̃ao de um comando, usa-se < fluxo de dados segue da direita para a esquerda.

E especialmente ́util para utilizar com comandos como o tr, que nao le arquivos diretamente.

O conteudo redirecionado por padrao ́e o de stdout = 1.

Para especificar stderr = 2 , usa-se 2> Para redirecionar ambos simultaneamente, usa-se &>

Standard Output - se o ficheiro file nao existir e criado
ls > file_list.txt
ls >> file_list.txt

Standard Input
sort processa ficheiro lista
sort < file_list.txt

sort processa ficheiro lista e depois gera novo ficheiro
sort < file_list.txt > sorted_file_list.txt



redirect standart error
ls -l > file.txt 2> errors.txt


redirect standard output and standard error
ls -l > files.txt 2>&1

atalho para o mesmo
ls -l >& files.txt

canalizar/pipes

para interligar dois comandos juntos para que a saida de um seja a entrada do proximo programa. dois ou mais comandos ligados desta forma, forma um tubo(pipe).

desenho de um pipe

Para realizar um pipe, colocar um caracter da barra vertical (|) entre dois comandos. Quando um programa recebe a contribuição de outro programa, executa alguma operação dessa entrada e escreve o resultado para a saída padrão(standart output), isto é referido como um filtro.

cat /proc/cpuinfo | grep "model name" | uniq

/proc/cpuinfo contem informacoes sobre o processador do computador. Seu conteudo foi canalizado com o comando cat /proc/cpuinfo para o comando grep "model name", que selecionara apenas as linhas contendo o termo model name. Por tratar-se de um computador com varios processadores, ha varias linhas "model name" iguais. A ultima canalizacacao e do comando grep "model name" para o comando uniq, que reduz linhas repetidas em sequencia para apenas uma ocorrencia.

Expressoes regulares

Expressoes regulares sao elementos de texto, palavras-chave e modificadores que formam um padrao, usado para encontrar e opcionalmente alterar um padrao correspondente. Muitos programas suportam o uso de expressoes regulares, e o comando grep é o mais comum para realizar buscas por eles, em textos. Vamos conhecer alguns caracteres e conjuntos que tem significado especial em expressoes regulares:

metacaracteres descricao
^ inicio de linha
$ padrao localizado no fim de linha
. corresponde a um caracter
* Corresponde a nenhuma ou mais caracteres
[caracteres] ou [numeros] qualquer caracter presente entre
conjunto descricao
[a-z] letra minúscula
[A-Z] letra maiscula
[a-zA-Z] corresponde a qualquer letra
[0-9] numeros
[a-zA-Z0-9] letras minúscula, maiscula e numeros

grep

Um dos uso mais comuns do comando grep e facilitar a inspeçao de ficheiros muito longos. Pode ser utilizado para exibir apenas as linhas que iniciem por um termo especıfico:

grep "^options" /etc/modprobe.d/alsa-base.conf

Como ocorre com a maior parte dos comandos, o caractere ^ permite utilizar a saıda de um comando como entrada de dados para o grep. A seguir, como utilizar colchetes para selecionar ocorrˆencias de qualquer um dos caracteres em seu interior:

fdisk -l | grep "^Disk /dev/sd[ab]"

opcao descricao
-c contar as linhas
-i ignora a diferença entre maisculas e minusculas
-f utilizar uma expressao regular contida num ficheiro
-n procurar somente na linha indicada
-v mostra todas as linhas exceto a que corresponder
[:alnum:] caracteres alfanumericos
[:alpha:] caracteres alfabeticos
[:digit:] caracteres numericos
[:upper:] caracteres alfabeticos maisculas
[:lower:] caracteres alfabeticos minusculos

egrep e fgrep

O comando egrep equivale ao comando grep -E, ele incorpora outras funcionalidades alem das expressoes regulares padrao. Por exemplo, com o egrep pode-se usar o operador pipe, que atua como o operador OU:

egrep "invençao|invençoes"

Serao retornadas todas as ocorrencias do termo invencao ou invencoes.

Ja o fgrep age da mesma forma que o grep -F, ele deixa de interpretar expressoes regulares. E especialmente util nos casos mais simples, em que o que se quer e apenas localizar a ocorrencia de algum termo simples. Mesmo se forem utilizados caracteres especiais, como $ ou ., estes serao interpretados literalmente, e nao pelo que representam numa expressao regular.
